Astro es un framework 'all-in-one' para construir sitios rápidos y basados en contenido.

# Principios de diseño de Astro

Podría estar un buen rato comentandoles acerca de las increíbles caracteristicas de Astro, pero dejemos eso para despues y comencemos por sus principios de diseño:

- **Basado en contenido**: Astro es excelente para construir sitios basados en contenido como documentación, blogs y portfolios. Aunque si búscamos construir una aplicación web interactiva más compleja quizá no es la mejor opción.
- **Server-first**: Prioriza el server-side rendering por sobre el client-side rendering siempre que sea posible. Al igual que Laravel o Rails, pero con la ventaja de qué todo sigue siendo HTML, CSS y JavaScript.
- **Velocidad**: El rendimiento es vital en muchos casos, y con muchos frameworks es normal experimentar tiempos de carga lentos. Con Astro podemos lograr un rendimiento increíble sin grandes optimizaciones de nuestra parte. 
- **Experiencia de Desarrollo**: Sentirse cómodo usando Astro es algo que pasa prácticamente desde el minuto uno. El lenguaje es un súperset de HTML, por lo que es muy familiar, pero tambien podemos utilizar expresiones JSX y se nos provee de CSS scoping por defecto.
Tambien podemos escribir componentes usando nuestro framework ó libreria favorita (React, Svelte, Vue, entre otros) 

# Conceptos principales

El objetivo principal de Astro es reducir la cantidad de JavaScript que envíamos al cliente. Y esto se logró gracias al concepto de **Islands**, que nos permite olvidarnos de todo ese código JavaScript innecesario.
